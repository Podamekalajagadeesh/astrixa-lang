Next.”# ASTRIXA Type System

ASTRIXA uses a strong, static type system with
type inference and optional annotations.

The goal is:
- Catch errors early
- Keep code readable
- Avoid runtime surprises

Key idea:
**Types should help, not punish.**

## Core Primitive Types

```ax
int      // integers
float    // floating-point
bool     // true / false
string   // UTF-8 strings
bytes    // raw binary
address  // blockchain address (native)
```

⚠️ `address` is native — this is already a Web3 advantage.

## Type Inference (Python-like ease)

```ax
let x = 10        // int
let y = 3.14      // float
let name = "AX"   // string
```

Compiler infers types automatically.

## Explicit Typing (when clarity matters)

```ax
let count: int = 0
let price: float = 9.99
```

## Immutability by Default (huge win)

```ax
let x = 10
x = 20    // ❌ compile-time error
```

Mutable:

```ax
var x = 10
x = 20    // ✅
```

This alone removes many bugs Python allows.

## Custom Types (structs)

```ax
struct Wallet {
    owner: address
    balance: float
}
```

## Optional & Nullable Values (no None chaos)

```ax
let name: Option<string> = none
let age: Option<int> = some(25)
```

Usage:

```ax
if name.is_some() {
    print(name.unwrap())
}
```

No unexpected `NoneType` crashes.

## Result Type (safe error handling)

```ax
Result<T> = ok(T) | error(string)
```

Example:

```ax
fn load_file(path: string) -> Result<string> {
    if not exists(path) {
        return error("File not found")
    }
    return ok(read(path))
}
```

Forces developers to handle errors.

## Generic Types (but simple)

```ax
struct Box<T> {
    value: T
}
```

Usage:

```ax
let b = Box<int> { value: 10 }
```

## Type Safety Rules (lock these)

- **No implicit type coercion**
- **No silent nulls**
- **No runtime type guessing**
- **All errors explicit**

These rules alone make ASTRIXA enterprise-safe.

## 🛑 STOP HERE

Do not:
- Add class inheritance
- Add dynamic typing
- Add macros
- Add operator overloading

**Minimal, strong core first.**
